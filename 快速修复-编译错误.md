# 快速修复 - Dashboard 编译错误

## 问题
Next.js 编译器报错，显示语法错误，但代码实际上是正确的。

## 解决方案

### 方法 1: 重启开发服务器 (推荐)

1. 按 `Ctrl+C` 停止当前的 `npm run dev`
2. 删除 Next.js 缓存:
   ```bash
   rm -rf .next
   # 或 Windows:
   rmdir /s .next
   ```
3. 重新启动:
   ```bash
   npm run dev
   ```

### 方法 2: 强制刷新

1. 确保文件已保存 (`Ctrl+S`)
2. 在浏览器中按 `Ctrl+Shift+R` 强制刷新
3. 等待自动重新编译

### 方法 3: 清理并重启

```bash
# 停止服务器 (Ctrl+C)
# 清理缓存
npm run clean
# 如果没有 clean 脚本，手动删除
rm -rf .next
rm -rf node_modules/.cache
# 重启
npm run dev
```

## 预防措施

1. 确保文件保存后再刷新浏览器
2. 大的代码改动后重启开发服务器
3. 保持 Next.js 版本更新
